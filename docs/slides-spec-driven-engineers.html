Stage Execution Framework

Every feature follows the same 3-phase workflow.

BEFORE

  • Run init-stage.sh
  • Creates proof template
  • Sets up state tracking

DURING

  • RED → GREEN → REFACTOR
  • Claude Code assists with TDD
  • Continuous test execution

AFTER Run quality gates, fill proof file, commit + tag

Every Stage: 3 Commands

# 1. BEFORE: Initialize stage (creates all files in correct locations)
./scripts/init-stage.sh --stage 9.7 --name "Transform DSL" --profile BACKEND_DOTNET

# 2. DURING: Implement with TDD (Claude Code assists here!)
# RED → GREEN → REFACTOR cycle

# 3. AFTER: Run gates then complete
./scripts/run-quality-gates.sh --stage 9.7 1 2 3 4 5 6 7 8
./scripts/complete-stage.sh --stage 9.7 --name "Transform DSL"

Scripts handle: directory structure, proof templates, CHANGELOG, git tags.

Advanced: Parallel Stage Execution

Work on multiple stages simultaneously with Git worktrees.

# Create worktrees for parallel stages
git worktree add ../project-stage-9.1 -b stage-9.1
git worktree add ../project-stage-9.2 -b stage-9.2
git worktree add ../project-stage-9.3 -b stage-9.3

# Each worktree = separate Claude Code session
# Artifacts isolated: stage-proofs/stage-9.1/, stage-proofs/stage-9.2/, etc.

# Merge in dependency order (not completion order!)
git checkout main
git merge stage-9.1 --no-ff
git merge stage-9.2 --no-ff
git merge stage-9.3 --no-ff

60% faster delivery when stages are independent.

Getting Started

  1. Create CLAUDE.md - Define your project spec, tech stack, constraints
  2. Break work into stages - Each stage = 1-2 weeks, testable deliverables
  3. Add the scripts - init-stage.sh, run-quality-gates.sh, complete-stage.sh
  4. Start Claude Code - It reads CLAUDE.md automatically
  5. Follow the workflow - BEFORE → DURING → AFTER

Framework is generic - works for .NET, TypeScript, or any stack.
